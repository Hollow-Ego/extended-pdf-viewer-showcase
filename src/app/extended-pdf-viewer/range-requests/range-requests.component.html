Range requests

Sometimes PDF files are large, even huge. By default, the PDF loader doesn't load the entire PDF file. Instead, it loads it in chunks. So it can display the first page while it's still loading the other pages.

Most of the time, you don't have to configure this feature (unless you want to switch it off). It just works. If it doesn't, check if your server supports range requests. If it does, the response headers have the attribute accept-ranges: bytes.

Vishal Patel has written a article range request and how to implement them in a simple node.js server. He's also published the source code on GitHub.

Getting rid of range requests and lazy loading

The default options disableStream and disableAutoFetch allow you to switch off the feature. In most cases, this means it takes longer until the PDF file is shown, so use this option only as a last resort.

Caveat: When I tried this, I didn't manage to disable range request. Chances are the feature is broken.
