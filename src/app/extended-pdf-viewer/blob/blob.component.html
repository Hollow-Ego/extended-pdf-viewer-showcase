If you're displaying a PDF file from a REST API, you often receive it as a BLOB. You can simply use the src attribute to display the BLOB. Since version 6.0.0, you can use this feature with a strict Content Security Policy strictly (see this issue for details).

This demo uses a file that's already in memory and a file that finishes loading after the PDF viewer has initialized. That's no difference from a programmers perspective, but it results in a different internal data flow, so both options are shown.

You can also download the PDF file as a BLOB. For instance, the user can upload a file, and you can use the method NgxExtendedPdfViewerService.getCurrentDocumentAsBlob() to send the file to your server. Or the user can fill a form embedded in the PDF file, and you can download the file containing the user's input.
