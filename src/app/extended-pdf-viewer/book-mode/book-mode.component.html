The book mode is intended to show a page-flip effect. Currently, this feature is under development. Truth to tell, it's possible it won't work reliable enough. Consider it a preview.

Problems to solve before making it an official features:

  • Book mode is a bit heavy on the CPU.
  • Rendering pages is unreliable. To enable smooth page turning, the two pages before and after the currently visible pages should always be rendered in advance. That doesn't always happen - possibly because of the high CPU usage, but I'm still looking for the root cause.

The PDF viewer supports the single-page mode. In this mode, you can scroll from page to page. Instead, you can only navigate to a new page by setting the page number or clicking a chapter in the sidebar.
